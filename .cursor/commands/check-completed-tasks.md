# 完了タスクのチェック（ソースコード正）

現在の**ソースコードを正**として、`docs/` 配下のタスクドキュメントのうち、実装済みのタスクにチェックを入れてください。

## 実行手順

### 1. 実装状況の確認

以下の判定基準で、リポジトリ内の**実装の有無**を確認すること。

- **ファイルの存在**: タスクで指定されているパス（例: `server/api/jobs/[id].get.ts`）にファイルが存在するか
- **設定の存在**: タスクが設定変更（例: `nuxt.config.ts` の routeRules / nitro）を求めている場合、該当設定が記述されているか
- **環境・手動タスク**: `.env` の有無や外部サービスの設定は**コードからは判定しない**。ファイルベースの実装タスクのみチェック対象とする。

確認する主なパス（例）:

- `server/api/**/*.ts`（API エンドポイント）
- `server/utils/*.ts`（ユーティリティ）
- `nuxt.config.ts`（routeRules, nitro, runtimeConfig 等）
- `app/composables/`, `app/pages/`, `app/stores/` 等、タスクに記載されたパス

### 2. 更新対象ドキュメント

次のファイルを対象に、**実装が確認できたタスクのみ**チェックを入れること。

| ファイル | チェック形式 | 備考 |
|----------|----------------|------|
| `docs/tasks/pending-tasks.md` | 未実装タスクを `- [ ]` → `- [x]` に変更。実装済みタスク（参考）セクションに不足があれば追記 | タスクID（例: 2.4.2, 4.1.4）とファイルパスで照合 |
| `docs/tasks/api-design-tasks.md` | `- [ ]` → `- [x]` | 同じタスクID・説明で照合 |
| `docs/tasks/setup-tasks.md` | `- [ ]` → `- [x]` | セットアップ項目は「ファイル・ディレクトリの存在」で判定。環境変数の中身や手動設定は未チェックのまま可 |

### 3. チェックを入れる条件

- **入れる**: 上記の確認で「ファイルが存在する」「必要な設定が含まれている」と判断したタスクのみ。
- **入れない**: ファイルが存在しない、設定が無い、または環境・手動設定のみのタスク。

### 4. 実装済みタスク（参考）セクション

`docs/tasks/pending-tasks.md` の「実装済みタスク（参考）」に、今回チェックを入れたエンドポイント・ファイルが含まれていなければ、その一覧に追記すること。

### 5. 注意事項

- 既存の文面やタスクIDは変更しない。チェック状態（`[ ]` / `[x]`）と実装済み一覧の追加のみ行う。
- 推測でチェックを入れない。ソースコード（または設定ファイル）で確認できたものだけを完了とする。
